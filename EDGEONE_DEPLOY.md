# Lumora EdgeOne Pages 部署指南

## 前置准备

1. **腾讯云账号**（已实名认证）
2. **Git 仓库**（GitHub / GitLab / Gitee 任选其一）
3. 视频文件已打包进 `public/videos/` 目录（共约 79MB）

## 部署步骤

### 1. 推送代码到 Git 仓库

把整个项目推送到你的 Git 仓库（GitHub / GitLab / Gitee 均可）：

```bash
git init
git add .
git commit -m "Lumora hero section with local videos"
git remote add origin <你的仓库地址>
git push -u origin main
```

⚠️ **注意**：`public/videos/` 目录共 79MB，已加入仓库。如果你的 Git 平台限制单文件 100MB（GitHub），单个视频最大 23MB，没问题。

### 2. 开通 EdgeOne Pages

1. 登录腾讯云控制台：https://console.cloud.tencent.com/
2. 搜索「EdgeOne」进入 EdgeOne 控制台
3. 左侧菜单选择「Pages」→「站点列表」
4. 点击「创建项目」→「导入 Git 仓库」
5. 授权并选择你的 Lumora 仓库

### 3. 构建配置

| 配置项 | 值 |
|--------|-----|
| 框架预设 | Next.js |
| 构建命令 | `npm run build` 或 `bun run build` |
| 输出目录 | `.next`（Next.js SSR 模式）或留空让 EdgeOne 自动识别 |
| 安装命令 | `npm install` 或 `bun install` |
| Node 版本 | 18.x 或 20.x |

### 4. 触发部署

点击「部署」，等待构建完成（首次约 2-3 分钟）。部署成功后会得到一个 `xxx.edgeone.app` 的免费域名。

### 5. 绑定自定义域名（可选）

在项目设置 → 域名管理中添加你自己的域名，按提示配置 CNAME 即可。

## CDN 加速效果

部署后，所有资源（HTML/CSS/JS/视频/字体）都通过 EdgeOne CDN 分发：

| 资源 | 大小 | 缓存策略 | 加速效果 |
|------|------|---------|---------|
| HTML | ~5KB | 不缓存（保证更新） | 边缘节点直出 |
| 视频（4个） | 79MB | 1年长期缓存 | 首次加载后秒开 |
| 字体 | ~14MB | 1年长期缓存 | 首次加载后秒开 |

**免费额度（个人项目足够用）：**
- 流量：10GB/月（约 130 次完整视频加载）
- 构建次数：1000 次/月
- 边缘节点：国内 2000+，海外 1000+

## 国内访问验证清单

部署后请在国内网络环境验证：
- [ ] 页面能正常打开（< 2 秒）
- [ ] 首个视频（金色时刻）能自动播放
- [ ] 切换到其他视频时无明显卡顿（首次切换需加载）
- [ ] 字体显示为衬线体（不是默认黑体）
- [ ] 移动端汉堡菜单正常
- [ ] 暗色模式（深林视频）正常切换

## 已知的国内访问优化

1. **视频按需加载**：首个视频 `preload="auto"`，其他 `preload="metadata"`，避免一次性下载 79MB
2. **智能切换**：切换视频时先检测是否已缓冲，已缓冲则立即切换，未缓冲则加载好再切（避免黑屏）
3. **加载指示**：切换加载中会显示小圆点动画
4. **超时兜底**：8 秒未加载完也会切换，避免一直卡在加载状态
5. **CDN 缓存**：`_headers` 配置视频 1 年缓存，二次访问秒开

## 常见问题

**Q: 部署后视频还是加载慢？**
A: 首次访问所有视频都要从源站拉取到 CDN 节点，第二次开始就秒开。可以部署后自己先把 4 个视频都点一遍预热。

**Q: 免费额度用完了怎么办？**
A: EdgeOne Pages 超额后会暂停服务（不会自动扣费）。可以等下月重置，或升级到付费版（按量计费，约 0.2 元/GB）。

**Q: 想要更好的视频体验？**
A: 可以考虑把视频上传到腾讯云点播（VOD）或 COS 对象存储，配合 CDN 加速。但免费额度有限，量大建议付费。
